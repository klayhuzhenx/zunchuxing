## 一、项目定位

**尊出行** 是华为 + 江淮（尊界品牌）+ 和行科技三方联合打造的高端出行服务平台，核心使命是把尊界品牌的高端出行服务做成一套标准化、线上化、可交付的体系。

覆盖两大场景：

| 场景 | 用户 | 核心诉求 |
|---|---|---|
| 企业公商务用车 | 企业员工（高管、行政、会务） | 准时、体面、企业结算 |
| 车主权益用车 | 尊界购车车主（C 端） | 打开即约、权益自动抵扣 |

> MVP 阶段仅包车出行 + 租车出行两种服务形态。预约出行、机场接送列入后续迭代。

## 三、技术栈与设计规范

### 3.1 技术栈

| 端 | 技术 |
|---|---|
| 乘客端原型 | 单文件 HTML + Tailwind CSS CDN + Material Symbols 图标 |
| 运营端（admin-app） | React 19 + TypeScript + Vite + Arco Design + React Router + Recharts |
| 司机端（规划） | 原生 App（iOS / Android） |
| 企业端（规划） | PC Web（技术栈待定） |

### 3.2 视觉设计规范

- **色彩**：尊界品牌色，深色主调（#000）搭配金色点缀（#D4AF37）
- **字体**：HarmonyOS Sans（中文）/ Hanken Grotesk（原型用）
- **图标**：Material Symbols（原型）/ 鸿蒙图标库（正式）
- **栅格**：8px 栅格系统，圆角统一 8–12px
- **层级**：页面层级 ≤ 3，核心操作 3 步内完成
- **状态反馈**：统一覆盖加载 / 成功 / 失败三种状态
- **无障碍**：支持大字体 + 屏幕朗读

### 3.3 写作约定（PRD 文档风格）

- 中文为主，专有名词保留英文
- 每章结构：业务说明 + 流程图（Mermaid）+ 字段表 / 列表 / 弹窗 / 详情 / 状态规则
- 业务术语用「」，强调用 **加粗**
- 状态颜色：待审核=琥珀色 / 已通过=绿色 / 已驳回=灰色 / 已禁用=红色

## 四、乘客端（微信小程序）功能全景

> 完整 PRD：[乘客端需求规格说明.md](./乘客端需求规格说明.md)
> HTML 原型：[系统原型设计/passenger-app.html](./系统原型设计/passenger-app.html)

### 4.1 注册登录

- **短信验证码登录**：无需密码，手机号 + 验证码即可。未注册自动注册并登录
- **微信快捷登录**：调起微信授权，首次需绑定手机号
- **安全机制**：极验（GeeTest）行为验证 + 短信频率限制（每手机号 10 次/小时、15 次/天；每 IP 30 次/天）
- **验证码有效期**：5 分钟，发送间隔 60 秒
- **登录态**：本地 Storage 存储，7 天有效期，距过期 1 天内自动刷新
- **协议**：登录前必须勾选《用户服务协议》和《隐私政策》

### 4.2 企业入驻

- **入口**：登录/注册页底部「企业入驻」链接
- **流程**：填写企业信息 → 上传营业执照 → 短信验证 → 提交审核 → 运营后台审核
- **关键规则**：一个手机号同一时间仅可持有一份审核中/已通过的入驻申请
- **审核结果**：通过（自动成为管理员）→ 驳回（可修改重提）
- **管理员密码**：入驻时设置，用于企业端 PC 后台登录

### 4.3 首页

- 服务 Tab 入口：包车出行 / 租车出行
- 企业身份用户展示所属企业名称
- 尊界车主展示「尊享权益」入口
- 底部导航：首页 / 行程 / 我的

### 4.4 包车出行

- **适用场景**：长时间/长途用车，跨城出差、多站点拜访、全天商务陪同
- **核心流程**：选择车型套餐 → 填写起止点和日期 → 确认下单 → 支付
- **套餐体系**：车型 + 套餐（如「增程星辉尊享版 · 尊享基础」）
- **计费**：套餐价 × 天数，日含 100km，超出 ¥5/km，超时 ¥100/小时
- **多日包车**：每日独立出行，可提前结束未开始日期按规则退款

### 4.5 租车出行

- **适用场景**：按天计费，自带或附司机
- **核心流程**：选择车型 → 填写取还车点和日期 → 添加驾驶人信息 → 确认下单 → 支付
- **计费**：日租价 × 天数，日含 200km，超出 ¥10/km，超时 ¥50/小时
- **送取车**：可选送车上门 / 上门取车服务
- **待取车子状态**：待送车（司机未送达）→ 已送达（等待乘客确认取车）

### 4.6 我的

- 个人信息：头像、昵称、手机号
- 身份标识：尊界车主标签 / 企业员工标签
- 功能入口：行程管理、电子发票、企业管理（管理员可见）、消息中心、联系客服、服务协议、隐私政策、退出登录

### 4.7 行程管理

- **状态 Tab**：全部 / 待支付 / 待派车 / 待开始 / 行程中 / 已完成 / 已取消
- **待补款子状态**：行程结束产生超里程/超时费用后进入待补款，差额支付后转为已完成
- **订单详情**：按包车/租车区分展示结构
  - 包车：行程信息 → 车型套餐 → 每日安排 → 费用明细 → 订单动态
  - 租车：取还车 → 车型 → 驾驶人 → 送取车 → 费用明细 → 订单动态
- **取消规则**：
  - 下单 20 分钟内或提前 4h+：免费取消
  - 距出发 < 4h：扣 25%；< 2h：扣 50%
  - 支付超时（20 分钟未支付）：自动取消，不扣费
  - 调度超时（出发前 2h 仍未派车）：自动取消，全额退款
- **退款**：个人支付原路退回；企业支付恢复额度；尊享权益出发 2h 外恢复次数
- **评价**：完成后可评价（司机/车辆/服务 1-5 星），匿名，不可修改

### 4.8 电子发票

- 个人发票：姓名 + 手机号
- 企业发票：企业名称 + 税号 + 接收邮箱
- 支持多订单合并开票
- 每笔订单仅可开票一次
- 状态：已开具 / 开票中 / 已作废
- 操作：下载 PDF、重新发送、申请作废重开、取消申请

### 4.9 企业管理（移动端）

- **管理员权限**：查看企业信息、管理员工（添加/删除）、查看额度及消费记录
- **员工权限**：查看企业基本信息、额度概览
- **添加员工**：需手机号已注册尊出行且未关联其他企业
- **额度记录**：消费（红色 -）/ 退款（绿色 +），支持按月筛选、按员工筛选

### 4.10 消息中心

- 消息类型：派车成功、行程开始/结束、取车/还车提醒、订单取消、感谢选择
- 未读角标：我的页面展示未读数，超 99 显示「99+」
- 相对时间：N 分钟前 / N 小时前 / N 天前
- 操作：点击查看详情 → 跳转关联订单 / 删除消息 / 全部已读

### 4.11 联系客服

- 当前阶段：电话客服 400-XXX-XXXX，7×24 小时
- 二期规划：接入微信客服 / 在线客服 H5

### 4.12 服务协议与隐私政策

- WebView 打开运营后台配置的 H5 页面
- 无需登录即可查看

## 五、运营管理后台 功能全景

> 完整 PRD：[运营端需求规格说明.md](./运营端需求规格说明.md)
> 代码仓库：[admin-app/](./admin-app/)

### 5.1 登录与安全

- **不开放注册**：账号由超管在「系统管理」创建
- **极验每次登录必过**：不再做「5 次失败后启用」
- **登录错误统一提示**「账号或密码错误」，避免账号枚举
- **忘记密码自助重置**：账号 + 手机号 → 极验 + 短信 → 设新密码
- **登录态**：8 小时，连续 30 分钟无操作自动失效
- **账号锁定**：连续失败 10 次锁定，需超管解锁
- **单端登录**：同一账号仅允许一处登录

### 5.2 工作台

- **核心数据卡片**（4 张）：今日订单、今日交易额、待派车、在线司机
- 卡片均可点击跳转对应模块筛选视图
- 数字下方展示较昨日对比（绿色 ↑ / 红色 ↓）
- **待办事项**：待派车、调度超时预警、待审核证件/企业、退款待执行、额度告急
- **数据趋势**：近 7 天订单量 + 交易额双折线图
- 数据每 60 秒自动刷新
- 按角色过滤可见内容

### 5.3 企业客户管理

- **企业来源**：小程序入驻申请 + 运营手动添加
- **基本信息仅 4 项**：企业名称 / 统一社会信用代码 / 联系人姓名 / 联系人手机号
- **新增企业弹窗**：填 4 项信息 → 直接「已通过」→ 以联系人手机号创建管理员 → 短信通知
- **审核**：通过（申请人成为管理员）/ 驳回（填写原因）
- **额度管理**：仅「调整额度」（调增/调减 + 必填原因），无充值入口
- **剩余额度筛选**：大于 / 小于 / 等于 + 金额
- **启用/禁用**：禁用后员工不可用企业支付，行程中订单照常

### 5.4 订单管理

- **一级 Tab**：订单 / 司机订单（一笔订单可对应多条司机订单）
- **状态自动流转**：运营不可手动改状态、不可主动取消
- **派车弹窗**：车辆筛选 + 月历视图 + 司机选择
- **冲突仅红色提示**，允许继续派车，不强制阻断
- **详情含**：下单人信息、关联司机订单、乘客备注（只读）、内部备注（运营写）
- **订单状态流**：待支付 → 待派车 → 待开始 → 行程中 → 已完成/已取消
  - 待补款：行程结束后产生额外费用的子状态
- **退款无「财务确认」步骤**，取消确认后直接退款

### 5.5 车辆管理

- 车辆列表：编号、车牌、车型、座位数、颜色、绑定司机、状态
- 车辆状态：在用 / 维修 / 已报废
- 车型从运营配置的车型库中选择
- 车辆位置暂未接入

### 5.6 司机管理

- 司机列表：编号、姓名、手机号、绑定车辆、状态、评分
- 司机状态：在线 / 离线 / 已禁用
- 司机审核：证件上传 → 运营审核

### 5.7 财务管理

- **企业账单**：按月汇总企业消费，支持导出 Excel
- **交易流水**：所有支付/退款/补款明细
- **退款**：按取消规则自动计算违约金后直接退款
- **补款**：订单差额补付记录
- 发票由第三方管理，运营端不直接操作

### 5.8 运营配置

- **计费规则**：按车型 + 套餐配置日租价、超里程单价、超时单价
- **车型管理**：车型库维护，含名称、座位数、图片
- **权益模板**：年度权益次数、有效期、抵扣规则
- **额度告急阈值**：低于阈值在工作台待办中提醒
- **电子围栏（运营区域）**：
  - 地图绘制多边形区域
  - 区域关联可调度车辆
  - 停用区域后不可下单和派车
  - 城市 → 区域的层级管理
- **协议管理**：服务协议和隐私政策的 H5 内容配置

### 5.9 数据分析与报表（⏳ 待写 PRD）

- 数据概览仪表盘：订单 / 营收 / 客单价 / 派车及时率等
- 订单报表：多维度筛选 + 导出
- 收入报表：按月/企业/订单类型汇总

### 5.10 系统管理（⏳ 待写 PRD）

- 账号管理：创建、编辑、停用/启用、重置密码
- 角色管理：超级管理员 / 运营管理员 / 财务管理员 / 客服管理员（四个预置角色）
- 权限矩阵：按角色精确控制模块可见性和操作权限
- 登录日志：记录所有登录行为，保留 180 天
- 操作日志：记录关键操作，含变更前后对比，保留 180 天
- 在线用户：当前在线列表 + 强制下线

## 七、核心业务规则速查

### 7.1 通用规则

| 规则 | 说明 |
|---|---|
| MVP 范围 | 仅包车 + 租车；预约出行/机场接送后续迭代 |
| 乘客端登录 | 短信验证码 + 极验，无密码 |
| 运营端登录 | 账号密码 + 极验每次必过，不开放注册 |
| 同一手机号 | 可同时绑定 C 端权益和 B 端员工身份 |
| 协议 | 登录前必须勾选，服务协议和隐私政策均可从登录页查看 |

### 7.2 支付与结算

| 规则 | 说明 |
|---|---|
| 个人支付 | 微信支付 |
| 企业支付 | 扣减企业总额度 |
| 补款 | 超时/超里程产生差额，订单进入「待补款」→ 补款链接支付 |
| 退款 | 取消/异常按规则退款，扣除违约金后余额退回 |
| MVP 无充值入口 | 乘客端/企业端均不开放充值，额度由运营后台调整 |

### 7.3 权益体系

- 购车赠送年度免费用车权益
- 下单时权益自动抵扣
- 出发 2h 外取消恢复权益次数，2h 内不恢复
- 调度超时自动取消不受此限制，全额恢复

### 7.4 取消与退款

| 场景 | 规则 |
|---|---|
| 下单 20 分钟未支付 | 自动取消，不扣费 |
| 出发前 2h 仍未派车 | 调度超时自动取消，全额退款（含权益恢复） |
| 主动取消（> 4h 出发） | 免费取消 |
| 主动取消（2-4h 出发） | 扣 25% |
| 主动取消（< 2h 出发） | 扣 50% |
| 待补款状态 | 不可取消，必须先完成补款 |

### 7.5 状态流转关键约束

- 运营**不可手动改订单状态**，全部系统自动流转
- 运营**不可主动取消订单**
- 派车冲突仅红色提示，**不强制阻断**
- 已禁用企业的行程中订单**照常进行**
