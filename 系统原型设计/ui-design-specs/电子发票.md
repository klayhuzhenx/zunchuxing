# 尊出行 · 电子发票模块 UI 设计说明

---

## 页面清单

> 本模块共 **4 张页面**：发票列表、选择开票订单、填写发票信息、发票详情。

### 7.1 发票列表

#### 页面内容

- 导航栏：返回按钮 + 标题"电子发票"
- 状态 Tab：全部 | 已开具 | 开票中
- 发票卡片列表，每张展示：发票类型标签（个人/企业）、抬头、金额、开具日期、状态（已开具/开票中/已作废）
- 底部"申请开票"按钮
- 空状态："暂无发票记录"

#### 交互行为

- 点击发票卡片 → 跳转发票详情页
- 点击"申请开票" → 跳转选择开票订单页
- 切换 Tab → 按状态筛选

---

### 7.2 选择开票订单

#### 页面内容

- 导航栏：返回按钮 + 标题"选择订单"
- 可开票订单列表（已完成且未开票），每笔展示：出行场景、行程摘要、金额，支持多选勾选
- 底部：已选笔数 + 合计金额 + "下一步"按钮（未勾选时置灰）

#### 交互行为

- 勾选订单 → 底部实时更新已选笔数和合计金额
- 点击"下一步" → 跳转填写发票信息页
- 多选订单 → 合并开票，发票内容按订单逐笔列出

---

### 7.3 填写发票信息

#### 页面内容

- 导航栏：返回按钮 + 标题"填写发票信息"
- 发票类型切换：个人 / 企业
- 个人发票字段：姓名、手机号
- 企业发票字段：企业名称、企业税号、接收邮箱
- 底部：已选笔数 + 合计金额 + "提交申请"按钮

#### 交互行为

- 切换发票类型 → 下方表单切换显示
- 企业身份用户 → 企业发票自动填入关联企业信息
- 点击"提交申请" → 校验必填字段 → 成功跳转发票列表
- 格式错误 → 对应字段下方提示

---

### 7.4 发票详情

#### 页面内容

- 导航栏：返回按钮 + 标题"发票详情"
- 发票状态 + 发票号
- 发票抬头：企业名称 + 税号（企业）/ 姓名 + 手机号（个人）
- 发票内容：出行场景 + 行程摘要 + 金额（合并发票按订单逐笔列出）
- 合计金额、开具日期
- 操作按钮："下载 PDF"、"重新发送至邮箱"
- 已开具状态额外操作："申请作废重开"
- 开票中状态额外操作："取消开票申请"

#### 交互行为

- 点击"下载 PDF" → 调起下载
- 点击"重新发送至邮箱" → Toast "已发送至 xxx@xxx.com"
- 点击"申请作废重开" → 弹窗确认 → 提交后台审核
- 点击"取消开票申请" → 弹窗确认 → 状态变为已取消，可重新申请

---

## 页面跳转关系

```
我的 → 发票列表
            ├──→ 选择开票订单 → 填写发票信息 → 返回列表
            └──→ 发票详情
                     ├── 下载 PDF
                     ├── 重新发送
                     ├── 申请作废重开
                     └── 取消开票申请
```
