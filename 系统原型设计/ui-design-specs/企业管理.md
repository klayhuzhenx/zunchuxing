# 尊出行 · 企业管理模块 UI 设计说明

---

## 页面清单

> 本模块共 **4 张页面**：企业管理首页、员工管理、添加员工、额度与消费记录。

### 8.1 企业管理首页

#### 页面内容

- 导航栏：返回按钮 + 标题"企业管理"
- 企业信息：企业名称、企业编号
- 额度概览：总额度、已使用、剩余额度
- 功能入口："员工管理"（仅管理员可见）、"额度与消费记录"

#### 权限说明

| 角色 | 权限 |
|---|---|
| 企业管理员 | 查看企业信息、管理员工（添加/删除）、查看额度及消费记录 |
| 企业员工 | 查看企业基本信息、查看额度总览 |

#### 交互行为

- 点击"员工管理" → 跳转员工管理页
- 点击"额度与消费记录" → 跳转额度与消费记录页

---

### 8.2 员工管理（仅管理员）

#### 页面内容

- 导航栏：返回按钮 + 标题"员工管理" + 右上角"+ 添加员工"按钮
- 员工列表，每项展示：姓名、角色（管理员/员工）、脱敏手机号
- 普通员工右侧展示"删除"入口，管理员不可删除自己
- 空状态："暂无员工"

#### 交互行为

- 点击"+ 添加员工" → 跳转添加员工页
- 点击"删除" → 弹窗确认 → 删除成功，从列表移除
- 管理员不可删除自己 → 提示"请联系运营后台处理"

---

### 8.3 添加员工

#### 页面内容

- 导航栏：返回按钮 + 标题"添加员工"
- 员工手机号输入框
- 提示文字："输入的手机号需已在尊出行注册，添加成功后员工小程序自动关联企业身份。"
- "确认添加"按钮

#### 校验规则

| 规则 | 错误提示 |
|---|---|
| 11 位中国大陆手机号 | "请输入正确手机号" |
| 须为已注册用户 | "该手机号未注册尊出行，请通知对方先注册" |
| 不可重复关联 | "该用户已关联其他企业" |

#### 交互行为

- 点击"确认添加" → 校验通过 → Toast "添加成功" → 返回员工列表
- 格式错误 → 对应字段下方提示

---

### 8.4 额度与消费记录

#### 页面内容

- 导航栏：返回按钮 + 标题"额度与消费记录"
- 筛选器："全部员工"下拉选择（仅管理员可见），员工仅看本人记录
- 记录列表，每项展示：变动类型（充值/消费/退款）+ 金额 + 时间 + 来源（充值：运营后台；消费/退款：操作人姓名 + 订单号）
- 支持分页加载

#### 变动类型

| 类型 | 标识 | 说明 |
|---|---|---|
| 充值 | 绿色 + 金额 | 运营后台为企业充值 |
| 消费 | 红色 - 金额 | 员工下单企业支付 |
| 退款 | 绿色 + 金额 | 订单取消后额度恢复 |

#### 交互行为

- 点击消费/退款记录 → 跳转关联订单详情
- 下拉筛选 → 切换员工视图
- 下拉加载更多 → 分页

---

## 页面跳转关系

```
我的 → 企业管理首页
           ├──→ 员工管理 → 添加员工
           └──→ 额度与消费记录 → 订单详情
```
